Отчет о проекте Freenit Backend

Основные принципы архитектуры

Проект Freenit Backend использует FastAPI для создания API, Ormar для работы с базой данных и Svelte для фронтенда. Архитектура проекта нечетко разделена на слои и компоненты, что затрудняет понимание и поддержку кода. Основные слои включают логический, данные и представление.

Используемые паттерны

  • Логический слой: FastAPI используется для создания API, но нет четкого разделения на сервисы и репозитории. Это приводит к смешиванию бизнес-логики и логики доступа к данным.
  • Слой данных: Ormar используется для работы с базой данных, но в файлах base_config.py и migration.py обнаружены проблемы с безопасностью и обработкой ошибок.
  • Представление: Svelte используется для фронтенда, но в проекте отсутствует разделение между фронтендом и бэкендом.

Комментарии по улучшению

  • Логический слой: Необходимо разделить бизнес-логику и логику доступа к данным. Используйте паттерн репозиторий для работы с базой данных и сервисы для бизнес-логики.
  • Слой данных: Исправьте проблемы с безопасностью и обработкой ошибок в файлах base_config.py и migration.py. Убедитесь, что все конфигурации и секреты хранятся в переменных окружения.
  • Представление: Разделите фронтенд и бэкенд на отдельные репозитории или модули для улучшения поддержки и понимания структуры проекта.

Примеры: В файлах base_config.py и migration.py обнаружены проблемы с безопасностью и обработкой ошибок. В base_config.py есть строки с жестко закодированными секретами, что является серьезной уязвимостью. В migration.py отсутствуют проверки на наличие конфигураций перед их использованием.

Пепе (разработчик)

Проект Freenit Backend представляет собой фреймворк для создания веб-приложений на основе FastAPI, Ormar и Svelte. Структура проекта сложная и неудобная для быстрого ознакомления новым разработчикам. В проекте отсутствует чёткое разделение компонентов, что затрудняет понимание и поддержку. Документация не полная и не содержит подробных инструкций по установке и развертыванию. В проекте используется Makefile для сборки, что является простым и понятным решением, но сам проект выглядит как перегруженный и сложноструктурированный. В целом, проект требует значительных усилий для поддержки и понимания, что делает его менее привлекательным для новых разработчиков.

Проект использует FastAPI и Ormar для создания API и управления базой данных, а также Svelte для фронтенда. Однако, структура проекта не соответствует принципам Unix-философии, где предпочтение отдается небольшим и независимым компонентам. Вместо этого, проект выглядит как монолитная структура, что затрудняет его поддержку и расширение. Кроме того, в проекте присутствуют проблемы с безопасностью, такие как хардкодированные секреты и потенциальные уязвимости SQL-инъекции.

Тесты присутствуют, но их структура также сложна и неудобна для понимания. В целом, проект требует значительных усилий для поддержки и понимания, что делает его менее привлекательным для новых разработчиков.
